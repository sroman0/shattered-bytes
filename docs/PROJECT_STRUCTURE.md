# Project Structure

This document summarizes how the repository is organized and where each kind of project material should live.

```text
shattered-bytes/
├── assets/                  # Source visuals and media used by gameplay and level generation
│   └── media/               # Source intro videos and related media
├── docs/                    # Project documentation, report, walkthrough, study material, and presentation
│   └── presentation/        # Beamer source, compile script, and generated presentation PDF
├── public/                  # Files served directly by Vite
│   ├── evidence/            # Recovered evidence images rendered by the Asset Viewer
│   ├── levels/              # Generated JSON levels loaded by the game
│   └── intro.mp4            # Intro video used by the intro component
├── scripts/
│   ├── level_generation/    # Official level generation scripts
│   └── experiments/         # One-off or historical scripts kept for reference
├── src/                     # React application source
│   ├── components/          # UI and gameplay components
│   ├── data/                # Campaign data, story data, signatures, and procedures
│   ├── hooks/               # Game state and orchestration hooks
│   └── utils/               # Hex, buffer, and crypto helpers
└── dist/                    # Production build generated by npm run build
```

## Practical Rules

Edit gameplay code in `src/`.

Edit campaign structure, briefings, objectives, procedure checks, and story data in `src/data/campaign.js`.

Regenerate official levels with:

```bash
npm run levels:build
```

Build the production version with:

```bash
npm run build
```

Do not manually edit `dist/` unless there is a specific deployment reason. It is generated output.

Keep walkthroughs, reports, and presentation material in `docs/`.

Keep temporary experiments under `scripts/experiments/`, not in the repository root.
